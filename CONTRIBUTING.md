# Contributing to This Portfolio

Thank you for your interest in contributing! Here are the guidelines:

## How to Contribute
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Commit your changes with descriptive messages
4. Push to your branch and open a Pull Request

## Code Style
- Follow PEP 8 for Python code
- Use meaningful variable names
- Add docstrings to all functions
- Include unit tests where applicable

## Project Standards
- All notebooks must include an introduction, EDA, modeling, and conclusion section
- README files must follow the existing template
- All code must be reproducible
